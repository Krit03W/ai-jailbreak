# 🚀 เกมแฮก AI ผู้พิทักษ์ฐานทัพดาวอังคาร

เว็บเกมกิจกรรม **AI Prompt Jailbreak** — มี **10 ห้องนิรภัย** แต่ละห้องมีรหัสลับของตัวเอง
ผู้เล่นเลือกห้องที่จะเจาะ แล้วหลอกล่อ AI (GUARDIAN) ให้ยอมบอกรหัสผ่านของห้องนั้น

## ▶️ วิธีรัน (เช้าพรุ่งนี้)

1. เปิด Terminal ที่โฟลเดอร์นี้
2. พิมพ์:
   ```
   node server.js
   ```
3. เปิดเบราว์เซอร์ตามลิงก์ที่ขึ้นในจอ (ปกติ `http://localhost:3000` ถ้าพอร์ตไม่ว่างจะเลื่อนไป 3001, 3002... ให้เอง)
4. ซูมจอ `Ctrl` + `+` ให้ตัวหนังสือใหญ่สำหรับฉายขึ้นจอใหญ่

> ต้องมี `OPENAI_API_KEY` ในไฟล์ `.env` (ใส่ไว้แล้ว) — ไม่ต้อง `npm install` อะไรทั้งสิ้น

## 🎮 วิธีเล่น

1. พิมพ์ **ชื่อกลุ่มที่กำลังเล่น** ในช่องบนสุด
2. เลือก **ห้องนิรภัย** 1 ใน 10 ห้อง (การ์ดจะบอกระดับความยาก 😀ง่าย / 😐กลาง / 😈ยาก)
3. เด็กพิมพ์ข้อความหลอกล่อ GUARDIAN กด Enter เพื่อส่ง
4. **ถ้า AI เผลอพูดรหัสของห้องนั้น** ระบบตรวจจับเอง → ขึ้นฉากฉลอง 🎉 + บันทึกว่ากลุ่มไหนเจาะได้ + การ์ดห้องเปลี่ยนเป็น "🔓 ถูกเจาะแล้ว"
5. กด **🔄 รีเซ็ตห้องนี้** เพื่อล้างบทสนทนาให้กลุ่มถัดไปเริ่มใหม่ในห้องเดิม หรือ **← เลือกห้องอื่น** เพื่อไปห้องใหม่

แต่ละห้องแยกกันอิสระ (บทสนทนา/สถานะเจาะไม่ปนกัน) เล่นหลายห้องพร้อมกันได้

## 🔐 หน้าแอดมิน (ตั้งรหัสจากหลังบ้าน)

คลิกปุ่ม **🔐 แอดมิน** มุมขวาบน → ใส่รหัสผ่านแอดมิน (ค่าจาก `ADMIN_PASSWORD`)

ในแผงแอดมินตั้งได้ทั้ง 10 ห้อง:
- **ชื่อห้อง**
- **รหัสลับ** ที่ AI ต้องปกป้อง
- **ระดับความยาก** (ง่าย/กลาง/ยาก)

กด **💾 บันทึกการตั้งค่า** — ค่าจะถูกเก็บลงไฟล์ `data/vaults.json` อยู่รอดแม้เซิร์ฟเวอร์รีสตาร์ท
ปุ่ม **♻️ รีเซ็ตสถานะเกมทุกห้อง** ล้างบทสนทนา + สถานะเจาะของทุกห้องรวดเดียว (ใช้ก่อนเริ่มกิจกรรมจริง)

> เคล็ดลับตั้งรหัส: ถ้าให้รหัส "ดูเป็นรหัสผ่าน" เช่นมีตัวเลข/ขีด (`NOVA-4242`) AI จะปกป้องแน่นกว่า
> ถ้าตั้งเป็นคำธรรมดา (`สับปะรด`) AI จะเผลอพูดง่ายขึ้น — ปรับความยากได้จากตรงนี้

## ⚙️ ตัวแปรตั้งค่า (environment variable)

ตั้งตอนรัน เช่น `ADMIN_PASSWORD="mypass" PORT=8080 node server.js`

| ตัวแปร | ค่าเริ่มต้น | ความหมาย |
|--------|------------|----------|
| `OPENAI_API_KEY` | — | คีย์ OpenAI (จำเป็น) |
| `ADMIN_PASSWORD` | `admin1234` | รหัสเข้าหน้าแอดมิน (**ควรเปลี่ยน**) |
| `MODEL` | `gpt-4o` | โมเดล OpenAI ที่ใช้ |
| `PORT` | `3000` | พอร์ต (เลื่อนอัตโนมัติถ้าไม่ว่าง) |
| `DATA_DIR` | `./data` | โฟลเดอร์เก็บค่าห้องนิรภัยที่แอดมินตั้ง |

## ✅ จุดเด่นด้านความปลอดภัยของเกม

- **รหัสไม่โผล่บนจอ** — เก็บเป็น system prompt จริงฝั่งเซิร์ฟเวอร์ API ฝั่งผู้เล่นไม่เคยส่งรหัสออกมา เด็กแอบอ่านจากจอไม่ได้
- **ตรวจจับการหลุดอัตโนมัติ** — normalize ก่อนเทียบ จับได้แม้ AI พยายามเลี่ยง เช่น เว้นวรรค `M A R S`, สะกดทีละตัว, พิมพ์ `mars2026`
- **หน้าแอดมินมีการยืนยันตัวตน** — endpoint ที่เห็น/แก้รหัสลับต้องมี token ที่ได้จากการล็อกอินเท่านั้น

## 🐳 Deploy บน Dokploy

ไฟล์ที่เกี่ยวข้อง: `Dockerfile`, `docker-compose.yml`, `.dockerignore`

1. Push โค้ดขึ้น Git repo (ไฟล์ `.env` + `data/` ถูก ignore ไว้แล้ว — คีย์ไม่หลุด)
2. ใน Dokploy: **Create Service → Compose** แล้วชี้มาที่ repo นี้
3. แท็บ **Environment** ใส่ตัวแปร:
   ```
   OPENAI_API_KEY=sk-....
   ADMIN_PASSWORD=รหัสแอดมินที่ตั้งเอง
   DOMAIN=jailbreak.example.com
   ```
   (ใส่ `MODEL` เพิ่มได้ถ้าต้องการ)
4. ชี้ DNS ของโดเมนมาที่ IP เซิร์ฟเวอร์ Dokploy แล้วกด **Deploy**

Traefik ของ Dokploy จะออกใบ SSL (Let's Encrypt) และ redirect HTTP→HTTPS ให้อัตโนมัติ ตาม label ใน `docker-compose.yml`
ค่าห้องนิรภัยที่แอดมินตั้งถูกเก็บใน volume `jailbreak-data` จึงอยู่รอดแม้ redeploy

> หมายเหตุ: certresolver ตั้งชื่อว่า `letsencrypt` (ค่าเริ่มต้นของ Dokploy) ถ้าติดตั้ง Dokploy ไว้ชื่ออื่น ให้แก้ใน `docker-compose.yml`

## 💡 เกร็ดสำหรับพิธีกร

- ก่อนเริ่มกิจกรรม: เข้าหน้าแอดมิน ตั้งรหัส/ความยากของแต่ละห้องตามใจ แล้วกด **♻️ รีเซ็ตสถานะเกมทุกห้อง** ให้สะอาด
- จัดห้องไล่ระดับ: ห้อง 1-3 ง่าย (ให้เด็กชิมลาง), 4-6 กลาง, 7-10 ยาก
- เทคนิคที่เด็กมักลอง: สวมบทบาท (roleplay), อ้างเป็นผู้ดูแลระบบ, ขอให้แปลภาษา/สะกดทีละตัว/ใส่ในบทกลอนหรือโค้ด
- ปิดท้ายด้วยบทเรียน: "อย่าเอาความลับไปฝากไว้ใน prompt" และ "AI ก็โดน social engineering ได้เหมือนคน"
